COMMIT MESSAGE FOR v1.1.0 RELEASE

Release v1.1.0: Path independence, configuration system, and comprehensive documentation reorganization

## Summary

v1.1.0 marks a major architectural improvement to persistent-ai-memory, introducing universal path independence and a professional configuration system. The system now works seamlessly on any deployment (standalone, OpenWebUI, Docker) with zero hardcoded paths. Complete documentation has been reorganized into five specialized guides for better usability.

## Major Changes

### üîß Path Independence (BREAKING CHANGE in structure, COMPATIBLE in function)
- **Removed all hardcoded paths** from core modules (short_term_memory.py, database_maintenance.py, etc.)
- **Environment variable driven configuration**: AI_MEMORY_DATA_DIR, AI_MEMORY_LOG_DIR, AI_MEMORY_EMBEDDING_URL, AI_MEMORY_LLM_URL
- **Smart path resolution**: Uses environment variables ‚Üí JSON config files ‚Üí sensible defaults
- **Cross-platform support**: Windows, Linux, macOS with proper path handling
- **Docker compatible**: Automatic path synchronization to /app/data paths in containers
- **Backward compatible**: Existing deployments continue to work with ~/.ai_memory/ defaults

### ‚öôÔ∏è Configuration System (NEW)
- **memory_config.json** - New file providing valve defaults for standalone/non-OpenWebUI deployments
  - LLM provider configuration (type, endpoint, model, API key)
  - Embedding configuration (provider endpoints)
  - Relevance thresholds and memory management settings
  - Memory bank configuration and display preferences
- **embedding_config.json** - Enhanced with multi-provider fallback support
  - Primary + fallback providers for high availability
  - Support for Ollama, LM Studio, OpenAI
  - Configurable timeouts, retries, batch processing
- **Configuration priority hierarchy**: Environment variables > JSON files > hardcoded defaults
- **Helper functions** in short_term_memory.py for smart config loading

### üè∑Ô∏è Tag Management System
- **tag_manager.py** ported from Friday project
- Automatic tag extraction, normalization, and canonicalization
- Integrated with database_maintenance.py for registry synchronization
- Supports local and Docker path registries
